# Release Coordination Runbook

1. Identify the previous and proposed release tags.
2. Collect merged changes between those tags.
3. Group user-visible changes into added, changed, fixed, and removed.
4. Flag breaking changes, migrations, and unresolved release blockers.
5. Produce release notes and a verification checklist.

Do not publish a release, push a tag, or message external users without
explicit approval.
